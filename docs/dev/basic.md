# 开发流程指南

## 开发流程

1. **Fork** 本项目到你的 GitHub 账户
2. 将项目克隆到本地开发环境
3. 创建新的功能分支，并切换到新分支
4. 在新分支上进行开发，并提交代码
5. 提交 **Pull Request** 到原项目的 `test` 分支并解决冲突
6. 若自动测试结果无异常则由维护者审核及合并

::: warning 重要提醒

1. 请先将代码 **Fork** 到自己的仓库，再进行开发
2. 请不要将代码直接提交到 `master` 分支
3. 请在提交 **Pull Request** 之前，先在本地进行测试，确保代码能够正常运行

:::
